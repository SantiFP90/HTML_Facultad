¿Por qué no solo divs?

  • Accesibilidad: Las etiquetas semánticas proporcionan información valiosa para los usuarios que dependen de lectores de pantalla y otras tecnologías de asistencia.
  • SEO (Optimización de Motores de Búsqueda): Los motores de búsqueda valoran la estructura semántica de una página web. El uso de etiquetas semánticas adecuadas puede mejorar la indexación y clasificación del contenido en los resultados de búsqueda.
  • Mantenimiento y Escalabilidad: Al utilizar etiquetas semánticas, tu código será más fácil de mantener y escalar a medida que tu proyecto crezca. La estructura clara facilita la comprensión del código, lo que simplifica las actualizaciones y modificaciones en el futuro.
  • Futuro-Proofing: El uso de etiquetas semánticas sigue las mejores prácticas de desarrollo web y te prepara para futuras actualizaciones y estándares. Además, demuestra un compromiso con la calidad y la accesibilidad del código.

Explicación del Código

Estructura Básica de HTML5

  • <!DOCTYPE html>: Define que el documento está usando HTML5.
  • <html lang="es">: Inicia el documento HTML y establece el idioma.
  • <head>: Contiene metadatos sobre el documento.
    • <meta charset="UTF-8">: Define la codificación de caracteres.
    • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Hace que el diseño sea responsivo.
    • <title>: Título de la página web.
  • <style>: Contiene los estilos CSS.

Semántica de las Etiquetas HTML5

  • <header>: Define el encabezado de la página o una sección. Suele contener títulos y navegación.
  • <nav>: Representa una sección de navegación con enlaces a otras páginas o partes de la misma página.
  • <main>: Delimita el contenido principal del documento. Debe ser único por documento.
  • <section>: Define secciones temáticas del contenido, como capítulos o agrupaciones de contenido relacionado.
  • <article>: Representa un contenido autónomo y autocontenido que podría ser distribuido o sindicado de forma independiente.
  • <aside>: Contiene información relacionada de manera tangencial con el contenido principal, como barras laterales o notas al margen.
  • <footer>: Define el pie de página de un documento o una sección, normalmente incluye información de autoría, enlaces relacionados, etc.
  • <form>: Contiene un formulario para la entrada de datos del usuario.
  • <input>: Define un campo de entrada de datos.
  • <textarea>: Define un campo de entrada de texto multilínea.
  • <button>: Representa un botón que puede ser clicado para enviar el formulario.
  • <audio>: Usado para incrustar contenido de audio.
  • <video>: Usado para incrustar contenido de video.
  • <canvas>: Elemento para gráficos mediante scripts (normalmente JavaScript).
  • <mark>: Resalta texto importante.
  • <small>: Muestra texto en tamaño de fuente más pequeño.
  • <time>: Representa una fecha y/o una hora en un determinado formato.
  • <meter>: Representa una medida escalar dentro de un rango conocido, o una fracción.
  • <output>: Representa el resultado de un cálculo o acción del usuario.
